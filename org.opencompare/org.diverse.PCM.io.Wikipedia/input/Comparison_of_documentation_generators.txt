<cite id="fn_4">[[#fn 4 back|Note (4):]]</cite> Ddoc has a macro system which can be customized to output any desired format. CHM, groff (manpages), XHTML, XML, and LaTeX (so PostScript and PDF) were tested.  They are not currently included in the standard distribution.  Standard HTML output also is generated using macros and can be redefined.

<cite id="fn_5">[[#fn 5 back|Note (5):]]</cite> Though not officially supported as an output format, Epydoc uses LaTeX and PostScript as intermediate steps to produce the final PDF documentation.

<cite id="fn_6">[[#fn 6 back|Note (6):]]</cite> Via [[Doclets]] from Third Parties.

<cite id="fn_7">[[#fn 7 back|Note (7):]]</cite> DOC++ outputs LaTeX and includes templates to enable conversion to PDF or PostScript with the appropriate latex tools. Required tools however are not included with DOC++.

<cite id="fn_8">[[#fn 8 back|Note (8):]]</cite> HyperSQL can output UnitTests (which can be embedded with the JavaDoc like description) as XML

<cite id="fn_9">[[#fn 9 back|Note (9):]]</cite> RDoc currently only provides generators for CHM and XML documents in the RDoc version provided as part of the Ruby 1.9 Core.

<cite id="fn_10">[[#fn 10 back|Note (10):]]</cite> RDoc generates documentation for [[RDoc|RI]], which is Ruby's version of the Unix man pages.

<cite id="fn_11">[[#fn 11 back|Note (11):]]</cite> Requires Microsoft HTML Help Workshop [http://msdn.microsoft.com/en-us/library/ms669985].
