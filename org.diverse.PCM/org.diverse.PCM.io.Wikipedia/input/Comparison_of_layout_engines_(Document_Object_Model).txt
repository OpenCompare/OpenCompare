The following tables compare [[Document Object Model|DOM]] compatibility and support for a number of [[layout engine]]s. Please see the individual products' articles for further information. Unless otherwise specified in footnotes, comparisons are based on the stable versions without any add-ons, extensions or external programs.

For features that are fully supported (based on DOM Level 2 or DOM Level 3 modules that are under W3C Recommendation), an exact version number is given if it is certain that the feature was added in such version. DOM Level 0 and DOM Level 3 modules that are still under development are not included.

== Explanation of the tables ==

===Values===
Values indicate the level of support in the most recent version of the layout engine, or (if a version number is given) in the specified version. Version numbers without any other value indicate the version at which the layout engine first fully supported the feature.

{| class="wikitable"
|-
! Value
! Meaning
|-
| style="background:#9F9;vertical-align:middle;text-align:center;" class="table-yes"|Yes
| Indicates that the layout engine fully supports this property/element when valid values are used.
|-
| style="background:#F99;vertical-align:middle;text-align:center;" class="table-no"|No
| Indicates that the property/element is completely ignored.
|-
| style="background:#FFB;vertical-align:middle;text-align:center; " class="table-partial"|Partial
| Indicates that the property/element is understood, but that not all values are supported. Supported values are implemented correctly.
|-
| style="background: #F90; color: black; vertical-align: middle; text-align: center; " class="incorrect"|Incorrect
| Indicates that the property/element is understood, but that it is not implemented correctly in all cases.
|-
| style="background: #DFDFFF; color: black; vertical-align: middle; text-align: center; " class="experimental table-experimental"|Experimental
| Indicates that the property/element is understood, but supported under an alternate name. May be incomplete or buggy.
|-
| style="background: #FED; color: black; vertical-align: middle; text-align: center; " class="dropped table-dropped"|Dropped
| Indicates that the property/element is no longer supported.
|-
| style="background: #fdd; color: black; vertical-align: middle; text-align: center; " class="nightly table-nightly"|[[Neutral build|Nightly build]]
| Indicates that the property/element is supported to some extent in an experimental/nightly build. Future support is expected.
|-
| style="background: #FED; color: black; vertical-align: middle; text-align: center; " class="depends table-depends"|Depends
| Indicates that the property/element is supported only on certain platforms, or if certain settings are configured.
|-
|}

===Trident Core (Fundamental) notes===
# <tt id="trident_Document_doctype">doctype</tt> &mdash; Prior to version 9.0, returns true when checked but always contains null.
# <tt id="trident_Document_getElementById">getElementById()</tt> &mdash; Prior to version 8.0 returned elements with <tt>name</tt> attribute set to value being found, not only <tt>id</tt>.
# <tt id="trident_Element_Attribute">getAttribute()</tt>, <tt>setAttribute()</tt>, <tt>removeAttribute()</tt></tt> &mdash; prior to version 8.0 returned, set, or removed, respectively, the corresponding DOM property instead of the string value of the attribute.
# <tt id="trident_Attr_specified">AttributeNode.specified</tt> &mdash; Does not set <code>specified</code> to <code>true</code> if ownerElement is null.
# <tt id="trident_isSameNode">Node.isSameNode</tt> &mdash; Prior to version 9.0, only supported for XML documents.

===Tasman Core (Fundamental) notes===
# <tt id="tasman_Attr_specified">AttributeNode.specified</tt> &mdash; Does not set <code>specified</code> to <code>true</code> if ownerElement is null.

===Gecko Core (Fundamental) notes===
# <tt id="gecko_Node_insertBefore">insertBefore(), replaceChild(), removeChild(), appendChild()</tt> &mdash; Fails on <code>Attr</code> nodes. <code>appendChild</code> does not work as expected for <code>DocumentFragment</code> nodes prior to 1.8.
# <tt id="gecko_Attr_specified">AttributeNode.specified</tt> &mdash; Prior to 1.9.1 does not set <code>specified</code> to <code>true</code> if ownerElement is null. Does not create Attribute Nodes for default values specified in the DTD.

===Presto Core (Fundamental) notes===
# <tt id="presto_Document_getElementById">getElementById()</tt> &mdash; Versions prior to Opera 9.50 returns elements with <tt>name</tt> attribute set to value being found, not only <tt>id</tt>.
# <tt id="presto_Attr_value">URI attribute value</tt> &mdash; In versions prior to Opera 9, all URI attribute values are resolved to full URI. Hence the value from <code>getAttribute()</code> and <code>getAttributeNode()</code> could be wrong if relative URIs are used.
# Attributes that are not supported in the layout engine are still parsed and available through DOM. E.g. char and charoff are available under .ch and .chOff.
# In Opera 8.54 methods DOMImplementation.createDocument and DOMDocument.createElementNS both fail to accept any non-empty namespace URI. They work in Opera 9 Beta 2 and they may also have worked before (somebody with Opera 8, please check.) The symptom is that the namespaceURI property is null/empty and serialization gives wrong result.

===WebKit Core (Fundamental) notes===
# <tt id="webkit_Attr_specified">AttributeNode.specified</tt> &mdash; Does not create Attribute Nodes for default values specified in the DTD.

===KHTML Core (Fundamental) notes===
# <tt id="khtml_Attr_specified">AttributeNode.specified</tt> &mdash; Does not create Attribute Nodes for default values specified in the DTD.

The second goal of the event model is to provide a common subset of the current event systems used in DOM Level 0 browsers. See also [[DOM Events]] for details.
